# MagSafe Dark

[English version](README.md)

MagSafe Dark — приложение для строки меню macOS, которое управляет индикатором на разъёме MagSafe.

Приложение умеет выключать индикатор, возвращать управление macOS, включать зелёный или оранжевый цвет, запускать временные эффекты, работать по недельному расписанию и показывать состояние задач Codex CLI.

> MagSafe Dark использует недокументированный SMC-ключ Apple `ACLC`. Обновление macOS или прошивки может изменить или отключить эту возможность.

## Требования

- macOS 13 или новее
- MacBook на Apple Silicon с MagSafe 3

Приложение проверено на MacBook Pro M5. После установки поддержку можно проверить командой:

```bash
magsafe-dark probe
```

Ожидаемый результат:

```text
supported
```

## Основные возможности

- Полностью выключить индикатор MagSafe.
- Вернуть обычное управление индикатором со стороны macOS.
- Включить зелёный, оранжевый цвет или один из режимов мигания.
- Запустить временный режим на заданное время.
- Настроить недельное расписание по дням и времени.
- Использовать отдельные режимы для работы, успеха и ошибки Codex.
- Запускать приложение автоматически при входе в систему.
- Открывать диагностику и журналы прямо из меню.

## Установка

Версия 1.4.0 пока устанавливается из исходников:

```bash
git clone https://github.com/bulava92/magsafe-dark.git
cd magsafe-dark
git checkout develop-1.4.0
zsh ./scripts/check-release.sh
zsh ./install.sh
```

Во время установки macOS один раз запросит пароль администратора. После установки обычная работа приложения и команд не требует `sudo`.

Приложение устанавливается сюда:

```text
/Applications/MagSafe Dark.app
```

## Использование приложения

Откройте **MagSafe Dark** из папки «Программы». Иконка появится в строке меню macOS.

В меню можно:

- выбрать режим индикатора;
- запустить или отменить таймер;
- настроить недельное расписание;
- настроить индикацию Codex;
- включить запуск при входе;
- выбрать вид иконки в строке меню;
- открыть диагностику и журналы;
- проверить наличие обновлений.

Горячие клавиши при открытом меню:

- `⌘⇧0` — переключить «Выключено» / «Штатный режим»;
- `⌘⇧G` — зелёный;
- `⌘⇧O` — оранжевый;
- `⌘Q` — выход.

## Режимы индикатора

| Режим | Что делает |
|---|---|
| Штатный | индикатором управляет macOS |
| Выключено | индикатор не горит |
| Зелёный | постоянный зелёный цвет |
| Оранжевый | постоянный оранжевый цвет |
| Одиночная индикация | короткая одиночная индикация |
| Медленное мигание | медленное мигание оранжевым |
| Быстрое мигание | быстрое мигание оранжевым |
| Мигание и выключение | мигание с последующим выключением |

## Таймеры

Таймер временно заменяет обычный режим индикатора.

Например, можно выключить индикатор на 30 минут. После окончания таймера приложение заново определит, какой режим должен действовать в этот момент. Оно не станет слепо восстанавливать устаревшее состояние.

Это важно, когда во время таймера меняется расписание:

```text
22:50  Запущен зелёный режим на 30 минут
23:00  Начался ночной интервал расписания
23:20  Таймер завершился
23:20  Применился ночной режим расписания
```

## Недельное расписание

Откройте пункт **«Настроить расписание…»** в меню приложения.

Для каждого интервала можно выбрать:

- один или несколько дней недели;
- время начала;
- время окончания;
- режим индикатора.

Интервал может переходить через полночь. Например, `23:00–08:00` начинается вечером и заканчивается утром следующего дня.

Стандартный шаблон:

```text
Каждый день  08:00–23:00  Штатный режим
Каждый день  23:00–08:00  Выключено
```

Шаблон изначально выключен. Чтобы он начал работать, включите расписание и нажмите «Сохранить».

Также можно выбрать, что делать вне заданных интервалов:

- использовать обычный режим macOS;
- держать индикатор выключенным;
- использовать последний постоянный ручной режим.

При включении расписания текущий режим применяется сразу. Уже запущенный таймер или активная индикация Codex при этом не прерываются.

### Ручное управление при активном расписании

Если выбрать режим вручную, он будет действовать до следующей границы расписания.

Пример:

```text
21:00  Вручную выбран зелёный режим
23:00  Начался следующий интервал расписания
23:00  Расписание снова получило управление
```

Когда расписание выключено, выбранный вручную режим остаётся активным, пока пользователь не изменит его сам.

## Индикация Codex CLI

MagSafe Dark может показывать через индикатор MagSafe состояние Codex CLI:

- идёт работа;
- задача завершилась успешно;
- задача завершилась с ошибкой.

Для каждого состояния можно выбрать отдельный режим. Для успеха и ошибки также настраиваются длительность и уведомления.

Запускайте Codex через обёртку:

```bash
codex-led
codex-led exec "Fix the failing tests"
```

Можно запускать несколько задач Codex одновременно. Рабочая индикация остаётся активной, пока работает хотя бы одна задача. Финальный результат показывается после завершения последней активной задачи.

## Как сочетаются расписание, Codex, таймер и ручное управление

Используется следующий приоритет:

```text
1. Пользовательский таймер
2. Активная индикация Codex
3. Ручной режим до следующей границы расписания
4. Текущий интервал расписания
5. Постоянный ручной режим
6. Обычный режим macOS
```

На практике это означает:

- таймер пользователя не перебивается Codex;
- Codex временно перекрывает расписание;
- после завершения Codex снова вычисляется актуальный режим расписания;
- ручной выбор при включённом расписании действует до следующего переключения расписания;
- при включении расписания старое ручное переопределение удаляется, и текущий интервал применяется сразу.

## Управление через Terminal

Прямое управление:

```bash
magsafe-dark off
magsafe-dark system
magsafe-dark green
magsafe-dark orange
magsafe-dark flash
magsafe-dark blink-slow
magsafe-dark blink-fast
magsafe-dark blink-off
```

Таймеры:

```bash
magsafe-dark for 900 off
magsafe-dark timer-status
magsafe-dark cancel-timer
```

Расписание:

```bash
magsafe-dark schedule edit
magsafe-dark schedule status
magsafe-dark schedule enable
magsafe-dark schedule disable
magsafe-dark schedule next
```

Запуск другой команды с индикацией:

```bash
magsafe-dark run -- make test
magsafe-dark run --working blink-slow -- npm run build
magsafe-dark run --success green --error blink-fast -- ./deploy.sh
```

Код завершения запущенной команды сохраняется.

## Диагностика

Полезные команды:

```bash
magsafe-dark status
magsafe-dark settings
magsafe-dark state
magsafe-dark diagnostics
magsafe-dark log-path
```

Диагностику и журналы также можно открыть из меню приложения.

## Решение проблем

### Индикатор не меняется

Проверьте совместимость:

```bash
magsafe-dark probe
```

Проверьте фоновую службу:

```bash
/usr/local/libexec/magsafe-led-client ping
/usr/local/libexec/magsafe-led-client status
```

Ожидаемый ответ на `ping`:

```text
pong
```

### Расписание не применяется

Проверьте его состояние:

```bash
magsafe-dark schedule status
```

Таймер и активная индикация Codex имеют приоритет выше расписания. Расписание применится после завершения временного состояния.

### Редактор расписания не открывается

Запустите его из Terminal, чтобы увидеть ошибку:

```bash
magsafe-dark schedule edit
```

## Удаление

Из папки проекта выполните:

```bash
zsh ./uninstall.sh
```

Скрипт вернёт индикатор под обычное управление macOS и удалит MagSafe Dark, фоновые службы, расписание, настройки и журналы.

## Конфиденциальность и безопасность

MagSafe Dark работает локально. Системная фоновая служба принимает только фиксированный набор команд управления индикатором и не выполняет произвольные команды оболочки.

После установки обычная работа приложения и CLI не требует `sudo`.
